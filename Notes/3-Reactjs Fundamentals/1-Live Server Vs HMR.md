أول شي في **Next.js** في شي اسمو **HMR (Hot Module Replacement)**.

### شو هو HMR؟

هو نظام بيخلي الصفحة **تتحدّث مباشرة بدون refresh كامل** لما تغيّر بالكود.

يعني مثلًا:

* تغيّر نص
* تغيّر style
* تغيّر component

الصفحة **تتحدث فورًا** بدون ما:

* تعمل refresh
* أو تخسر الstate أحيانًا.

وهيدا جزء من بيئة التطوير في **React** و **Next.js**.

---

### طيب شو وضع Live Server مع HTML؟

إضافة **Live Server** بتعمل شي مشابه لكن **مش نفس الشي**.

#### Live Server

* لما تغيّر بالـ HTML / CSS / JS
* يعمل **auto refresh للصفحة**.

يعني:

```
save → refresh كامل للصفحة
```

#### HMR

أما HMR يعمل:

```
save → يبدّل الجزء المتغير فقط
بدون refresh كامل
```

---

### الفرق السريع

| الميزة                 | Live Server | HMR          |
| ---------------------- | ----------- | ------------ |
| Refresh كامل           | نعم         | لا           |
| يبدل جزء من الصفحة فقط | لا          | نعم          |
| يحافظ على state        | لا          | أحيانًا      |
| يستخدم مع              | HTML عادي   | React / Next |

---

### مثال سريع

لو عندك component في **Next.js**

```jsx
export default function Page() {
  return <h1>Hello</h1>;
}
```

إذا غيرت:

```jsx
return <h1>Hello Omar</h1>;
```

HMR بيحدث النص **مباشرة** بدون reload.

---

✅ الخلاصة:

* **Live Server = auto refresh**
* **HMR = تحديث ذكي بدون refresh**

---

إي طبيعي يصير معك هالشي 👍 وكتير ناس بتفكر إنو في مشكلة.

خليني خبرك ليش أحيانًا ما بتشوف التغيير فورًا.

### 1️⃣ التغيير ما كان بالـ Component نفسه

في **Next.js** نظام **Fast Refresh** (تطوير لـ HMR) أحيانًا ما يقدر يحدث الصفحة إذا التغيير مش مباشر بالـ component.

مثال:

```jsx
const name = "Omar";

export default function Page() {
  return <h1>{name}</h1>;
}
```

إذا غيرت `name` أحيانًا ما يحدث فورًا.

---

### 2️⃣ غيرت ملف ما مربوط مباشرة بالصفحة

مثل:

* utils
* constants
* config

مثال:

```js
// utils.js
export const siteName = "My Site";
```

إذا غيرته أحيانًا لازم refresh.

---

### 3️⃣ في error صغير بالكونسول

أحيانًا **React** ما يعمل refresh إذا في error.

شوف:

```
F12 → Console
```

---

### 4️⃣ الكاش

أحيانًا المتصفح يحتفظ بالنسخة القديمة.

الحل:

```
Ctrl + F5
```

---

### 5️⃣ dev server علّق

أحيانًا السيرفر تبع **Next.js** يعلق.

الحل:

```
Ctrl + C
npm run dev
```

💡 نصيحة مهمة للمبتدئين:

حتى المطورين الكبار أحيانًا يعملوا **refresh يدوي** 😂
فمش مشكلة إذا عملت refresh.

---

إي، لما تعمل **refresh** كامل للصفحة، المتصفح يتصرف كأنك فتحت الموقع من جديد. خليني أوضح الفرق 👇

### لما تعمل Refresh عادي

المتصفح يعمل:

1. يطلب الصفحة من السيرفر من جديد.
2. يحمل **HTML**.
3. يحمل **CSS**.
4. يحمل **JavaScript**.
5. يعيد تشغيل التطبيق كله.

يعني كل شيء يبدأ من الصفر.

لهيك في **React** أو **Next.js**:

* كل **state** يرجع للبداية
* كل **data** يمكن تنطلب من جديد
* كل **components** يعاد رسمها.

---

### أما مع HMR / Fast Refresh

هنا الفكرة مختلفة.

بدل ما يعيد تحميل كل الموقع، النظام يعمل:

* يحدد **أي ملف تغير**
* يحدث **هذا الجزء فقط**

مثال:

قبل التعديل:

```jsx
export default function Page() {
  return <h1>Hello</h1>;
}
```

بعد التعديل:

```jsx
export default function Page() {
  return <h1>Hello Omar</h1>;
}
```

مع **Fast Refresh** في **Next.js**:

* لا يعيد تحميل الموقع كامل
* فقط يغير النص في الـ component.

---

### الفرق بسرعة

| الحالة             | ماذا يحدث                |
| ------------------ | ------------------------ |
| Refresh عادي       | يعيد تحميل كل الموقع     |
| HMR / Fast Refresh | يغير الجزء الذي تغير فقط |

---

💡 مثال واضح:

لو عندك **form** وكتبت فيه بيانات:

* مع **refresh** → البيانات تختفي
* مع **HMR** → البيانات تبقى غالبًا

---

إذا حاب، فيني كمان أشرحلك **كيف يكتشف Next.js الملف الذي تغير خلال أقل من ثانية** (الفكرة وراء HMR). هاي من الأشياء الحلوة التي يفهمها المطور بعد ما يبدأ بـ **Next.js**.

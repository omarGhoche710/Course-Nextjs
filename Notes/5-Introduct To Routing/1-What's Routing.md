## 🧠 شو هو Routing بـ Next.js؟

الـ **Routing** يعني:

> كيف المستخدم بيتنقّل بين صفحات الموقع باستخدام الـ URL

مثال:

* `/` → الصفحة الرئيسية
* `/about` → صفحة "عنّا"
* `/contact` → صفحة التواصل

---

## ⚡ المميز بـ Next.js

بـ Next.js ما فيك تكتب routes بإيدك مثل React ❌
ما في:

```js
<Route path="/about" />
```

بدل هيك في فكرة:

> 🧩 **File-Based Routing**

---

## 📁 يعني شو File-Based؟

يعني:

> اسم الملف + مكانه = الرابط (URL)

---

## 🔥 مثال سريع

لو عندك هيك:

```
app/
 ├─ page.tsx
 ├─ about/
 │   └─ page.tsx
```

### النتيجة:

* `app/page.tsx` → `/`
* `app/about/page.tsx` → `/about`

---

## 🧠 الفكرة الأساسية

Next.js بيقولك:

> "إنت بس اعمل folders و files… وأنا بحوّلهم لروابط"

---

## 🎯 خلاصة

* Routing = التنقّل بين الصفحات
* بـ Next.js = بيصير أوتوماتيك
* بيعتمد على الملفات (file-based)

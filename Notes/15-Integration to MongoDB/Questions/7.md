# What are the best practices for handling MongoDB ObjectIds in Next.js?

MongoDB ObjectIds are not serializable by default in Next.js props. Best practices include:

- Convert ObjectIds to strings using _id.toString()
- Use JSON.parse(JSON.stringify(data)) to serialize entire objects
- Create a helper function to sanitize MongoDB documents before returning them
- When querying by ID, convert string IDs back using new ObjectId(id)

**3. عمل helper function للتحويل:**

بدل ما تكتب التحويل بكل مكان، تعمل function واحدة:

```ts
// lib/serialize.ts
export function serialize(data: any) {
  return JSON.parse(JSON.stringify(data))
}
```

وبتستخدمها هيك:

```ts
const users = await User.find()
return Response.json(serialize(users)) // ✅ كل شي اتحول تلقائياً
```

---

**4. لما بدك تجيب document بالـ ID:**

المستخدم بيبعتلك الـ ID كـ string من الـ URL — بس MongoDB بده ObjectId مش string:

```ts
import { ObjectId } from "mongodb"

// ❌ غلط — id هون string
await User.findById("648a1f2e3c4d5e6f7a8b9c0d")

// ✅ صح — حوّله لـ ObjectId
await User.findById(new ObjectId("648a1f2e3c4d5e6f7a8b9c0d"))
```

**مثال كامل:**

```ts
// app/api/users/[id]/route.ts
export async function GET(req, { params }) {
  await connectDB()

  const user = await User.findById(new ObjectId(params.id))
  return Response.json(serialize(user)) // ✅ حوّل الـ ObjectId لـ string
}
```

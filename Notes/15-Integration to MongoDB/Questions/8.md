## Authentication Methods in Next.js

### When to use each?

| السؤال | الاختيار |
|--------|----------|
| بدي login بـ Google/GitHub؟ | NextAuth |
| عندي mobile app؟ | JWT |
| web only + email/password؟ | iron-session |

---

### NextAuth — with vs without MongoDB Adapter

| | بدون Adapter | مع Adapter |
|---|---|---|
| الـ session محفوظة | بـ cookie | بـ MongoDB |
| تشوف المستخدمين بـ DB | ❌ | ✅ |
| الأبسط | ✅ | ❌ |

```ts
// بدون Adapter — session بـ cookie بس
export const { handlers, auth, signIn, signOut } = NextAuth({
  providers: [GitHub],
})

// مع Adapter — session و user بـ MongoDB
export const { GET, POST } = NextAuth({
  adapter: MongoDBAdapter(clientPromise),
  providers: [GoogleProvider({...})]
})
```

### ⚠️ Sign in by GitHub لا يحفظ password

GitHub بيعطيك بس:
```json
{
  "name": "John",
  "email": "john@gmail.com",
  "image": "https://github.com/...",
  "password": null
}
```

الحل — تتحقق قبل login:
```ts
const user = await User.findOne({ email })

if (!user.password) {
  return Response.json({ 
    error: "هيدا الحساب مسجل بـ GitHub — اضغط Sign in with GitHub" 
  })
}
```

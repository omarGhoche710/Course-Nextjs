### 📝 ملاحظة: Session Data vs Persistent Data

هناك نوعان من البيانات في التطبيقات: **بيانات مؤقتة** و**بيانات دائمة**. فهم الفرق بينهما مهم عند التعامل مع **Sessions** و**Cookies** و**localStorage**.

---

## 1️⃣ Session Data (بيانات الجلسة)

* بيانات **مؤقتة** يتم إنشاؤها بعد تسجيل الدخول.
* تُستخدم للتحقق من هوية المستخدم (**Authentication**).
* **تنتهي** عند:

  * تسجيل الخروج (Logout)
  * انتهاء مدة الجلسة (Session Timeout)
  * أحيانًا عند إغلاق المتصفح

**أمثلة:**

* `sessionId`
* معلومات المستخدم بعد تسجيل الدخول
* صلاحيات الوصول (Permissions)

---

## 2️⃣ Persistent Data (بيانات دائمة)

* بيانات **تبقى محفوظة** حتى بعد انتهاء الجلسة أو إغلاق المتصفح.
* تُستخدم لتخصيص تجربة المستخدم.
* لا تتأثر عادةً بالـ Logout أو Session Timeout.

**أمثلة:**

* **Theme** (Light / Dark mode) 🌙
* **Language** (اللغة) 🌍
* تفضيلات المستخدم (User Preferences)
* خيار "Remember me"

---

## الفرق السريع

| الخاصية           | Session Data | Persistent Data |
| ----------------- | ------------ | --------------- |
| المدة             | مؤقتة        | طويلة / دائمة   |
| تنتهي عند Logout  | نعم          | لا              |
| تنتهي عند Timeout | نعم          | لا              |
| مثال              | sessionId    | theme, language |

---

### مثال بسيط للتوضيح

* المستخدم سجّل دخول → تم إنشاء **Session**
* المستخدم اختار **Dark Theme**
* بعد 30 دقيقة بدون نشاط → انتهت **Session**
* لكن عند الدخول مرة أخرى → **Theme** بقي محفوظ ✅

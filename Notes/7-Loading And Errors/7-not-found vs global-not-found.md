## 📊 جدول مقارنة

| العنصر | `not-found.tsx` | `global-not-found.tsx` |
|--------|-----------------|-------------------------|
| **المكان** | داخل segment أو route معيّن (مثلاً: `app/users/[id]/not-found.tsx`) | مباشرة تحت جذر المشروع `app/global-not-found.tsx` |
| **متى يشتغل؟** | أوتوماتيكياً لما تستدعي `notFound()` من `next/navigation` | أوتوماتيكياً لما المستخدم يدخل URL ما إله أي route |
| **نوع الحالة** | غياب بيانات داخل صفحة موجودة | غياب المسار نفسه بالكامل |
| **السلوك** | بيعمل render لواجهة مخصّصة داخل نفس الـ route | بيرجع صفحة HTML كاملة قبل ما يبدأ الـ React render |
  | **العقلية** | "Absence is not failure" | "This route never existed" |
  | **التعليمية** | 404 مش crash، هو حالة متوقعة | بيصير قبل ما التطبيق يبلّش يشتغل |

  ---

  ## 🧠 الخلاصة
  - `not-found.tsx` → **404 داخلي** مرتبط بالـ route لما البيانات ناقصة.  
  - `global-not-found.tsx` → **404 خارجي** مرتبط بالـ router لما الرابط نفسه مش موجود.

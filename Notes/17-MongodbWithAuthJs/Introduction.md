
---

**الفكرة:**

```
بدل ما المستخدم يدخل email + password
→ بتوجهه لـ GitHub
→ GitHub بيتحقق منه
→ بيرجعه لتطبيقك مع معلوماته
```

---

**ليش مختلف؟**

```
email/password:
→ أنت بتتحقق من الـ password
→ أنت بتعمل الـ session

GitHub login:
→ GitHub بيتحقق
→ GitHub بيبعتلك معلومات المستخدم
→ أنت بس بتحفظهم وبتعمل session
```

---

**وهون بيجي دور AuthJS —**

بدل ما تعمل كل هيدا بنفسك، AuthJS بيتعامل مع كل شي تلقائياً:

```
1. بيوجه المستخدم لـ GitHub
2. بياخد معلوماته
3. بيحفظه بـ MongoDB
4. بيعمل الـ session
```

## 📝 Cookie Notes — Next.js

### 🍪 ما هي الكوكي؟
- بيانات صغيرة يخزنها المتصفح.
- تُرسل تلقائيًا مع كل request للسيرفر.
- تستخدم لحفظ: **session**, **theme**, **token**, **language**.

---

### ⚙️ إنشاء كوكي في Next.js
```ts
response.cookies.set("myCookie", "cookieValue", {
  path: "/admin",
  httpOnly: true,
  secure: true,
});
```

- `path` → يحدد وين الكوكي مسموح تنبعت داخل الموقع.  
- `httpOnly: true` → تمنع الوصول للكوكي من الـ JavaScript (تحمي من XSS).  
- `secure: true` → الكوكي تُرسل فقط عبر HTTPS (تحمي من التسريب).  

---

### 📏 Rule — Cookie `path`
- الكوكي تُرسل فقط للـ routes اللي تبدأ بنفس قيمة `path`.  
- إذا `path = "/"` → الكوكي متاحة على كل الموقع. 🌐  
- إذا `path = "/something"` → الكوكي متاحة فقط داخل `/something` وما يتفرّع منها.  

---

### 📖 قراءة الكوكي
- **Server (Route Handler):**
```ts
import { cookies } from "next/headers";

const cookieStore = cookies();
const value = cookieStore.get("myCookie");
```

- **Client (Browser):**
```ts
document.cookie
```

- **DevTools:**
```
F12 → Application → Cookies
```

---

### 🔐 خصائص الأمان
- `httpOnly` → تمنع الوصول من JavaScript.  
- `secure` → تُرسل فقط عبر HTTPS.  

---

## 🎯 ملخص سريع
- **Cookie** = بيانات صغيرة محفوظة بالمتصفح.  
- **`response.cookies.set()`** = إنشاء أو تعديل كوكي.  
- **`path`** = يحدد نطاق الكوكي داخل الموقع.  
- **`httpOnly` + `secure`** = حماية إضافية للكوكي.  
- **قراءة الكوكي** → عبر `cookies().get()` بالسيرفر أو `document.cookie` بالمتصفح.

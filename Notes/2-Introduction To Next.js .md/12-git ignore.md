هذا الملف اسمه **`.gitignore`** ويُستخدم مع **Git**.

وظيفته:
📌 إخبار Git **أي ملفات لا يجب رفعها إلى المستودع (repository)**.

---

# الفكرة الأساسية

عندما تكتب:

```bash
git add .
```

يقوم **Git** برفع كل الملفات…
إلا الملفات الموجودة في **`.gitignore`**.

---

# مثال مهم

```text
/node_modules
```

مجلد **node_modules** يحتوي آلاف الملفات التي ينزلها **npm**.

لذلك لا نرفعها إلى GitHub.

بدل ذلك نرفع فقط:

```
package.json
package-lock.json
```

وعندما يشغل شخص المشروع يكتب:

```bash
npm install
```

فيُعاد تحميلها.

---

# شرح أهم الأقسام

## dependencies

```text
/node_modules
```

يتجاهل مكتبات المشروع.

---

## next.js

```text
/.next/
/out/
```

هذه ملفات يبنيها **Next.js** تلقائيًا عند تشغيل المشروع.

---

## production

```text
/build
```

مجلد يحتوي النسخة النهائية للموقع.

---

## env files

```text
.env*
```

ملفات **البيئة** التي تحتوي غالبًا على:

* API keys
* كلمات مرور
* إعدادات حساسة

لهذا لا يجب رفعها.

---

## debug logs

```text
npm-debug.log*
yarn-error.log*
```

هذه ملفات أخطاء تُنشأ عند حدوث مشاكل في **npm**.

---

## system files

```text
.DS_Store
```

ملف ينشئه نظام **macOS** تلقائيًا.

---

# الخلاصة

`.gitignore` يخبر **Git**:

❌ لا ترفع هذه الملفات
✅ ارفع فقط الملفات المهمة للمشروع

مثل:

* الكود
* `package.json`
* إعدادات المشروع

---

💡 معلومة مهمة جدًا للمطورين:
أكبر خطأ يقع فيه المبتدئون هو **رفع مجلد `node_modules` إلى GitHub** — وهذا قد يجعل المشروع حجمه مئات الميغابايت 😅.

ملف **`package-lock.json`** هو ملف ينشئه **npm** تلقائيًا عندما تكتب:

```bash
npm install
```

وظيفته الأساسية:
🔒 **تثبيت (lock) النسخ الدقيقة للمكتبات** حتى يعمل المشروع بنفس الطريقة عند الجميع.

---

# الفكرة الأساسية

في `package.json` قد ترى:

```json
"tailwindcss": "^4"
```

الرمز `^` يعني:

> يمكن تثبيت أي نسخة من 4.x.x

مثال:

* 4.0.0
* 4.1.2
* 4.5.7

لكن المشكلة:
قد يحصل كل شخص على **نسخة مختلفة**.

هنا يأتي دور **package-lock.json**.

---

# ماذا يفعل package-lock.json

يقوم بتسجيل **النسخة الدقيقة** لكل مكتبة.

مثال:

```json
"tailwindcss": {
  "version": "4.0.3"
}
```

وهكذا:

حتى لو كتب شخص آخر:

```bash
npm install
```

سيحصل على **نفس النسخة تمامًا**.

---

# لماذا هذا مهم؟

بدونه قد يحدث:

👨‍💻 يعمل المشروع عندك
❌ لا يعمل عند زميلك

لأن نسخة المكتبة مختلفة.

---

# الفرق بين الملفين

| الملف               | وظيفته                        |
| ------------------- | ----------------------------- |
| `package.json`      | يحدد المكتبات المطلوبة        |
| `package-lock.json` | يحدد النسخة الدقيقة لكل مكتبة |

---

# هل يجب رفعه إلى Git؟

نعم ✅
يجب رفعه إلى **Git** حتى يحصل كل الفريق على نفس النسخ.

---

# الخلاصة

`package-lock.json` هو:

* ملف ينشئه **npm**
* يثبت نسخ المكتبات
* يجعل المشروع يعمل بنفس الطريقة عند الجميع

---

💡 معلومة مهمة:
في المشاريع الكبيرة مثل **Next.js** قد يحتوي `package-lock.json` على **آلاف الأسطر** لأن كل مكتبة لها dependencies خاصة بها.

# 1️⃣ ESLint Flat Config + next lint removal

### شو تغيّر:

* **ESLint صار يستخدم flat config** بشكل افتراضي بدل `.eslintrc.js` أو `.json`.
* الأمر **`next lint` أُزيل**.
* **next build** لم يعد يقوم بالـ linting تلقائيًا.
* خيار `eslint` في `next.config.ts` تم حذفه.

### شو تعمل الآن:

* استخدم **ESLint CLI مباشرة** في `package.json`:

```json id="esl1"
{
  "scripts": {
    "lint": "eslint . --ext .js,.jsx,.ts,.tsx"
  }
}
```

* وملف الـ config صار `eslint.config.mjs`:

```ts id="esl2"
import { FlatCompat } from "@eslint/eslintrc";

export default {
  root: true,
  parser: "@typescript-eslint/parser",
  plugins: ["@typescript-eslint"],
  rules: {
    "no-console": "warn",
    "semi": ["error", "always"]
  },
};
```

> كل شيء صار يدار عبر **ESLint CLI** بدل أوامر Next.js.

# 2️⃣ Scroll-behavior override

### التغيير:

* Next.js **لم يعد يفرض scroll instant** عند التنقل بين الصفحات في SPA.
* لو بدك ترجع السلوك القديم (smooth scroll)، أضف **attribute** على `<html>`:

```tsx id="scroll1"
<html lang="en" data-scroll-behavior="smooth">
```

* في المشروع الموجود، تم إضافته في:

```ts
app/layout.tsx
```

# 3️⃣ Performance and output changes

### أهم النقاط:

1. **Dev / Start commands أسرع** مع تحسين UX في التيرمينال.
2. **Build output** لم يعد يعرض:

   * Size metrics
   * First Load JS metrics

> السبب: هذه metrics كانت **مضللة مع React Server Components (RSC)**.

> باختصار: الأداء صار أفضل والمخرجات أنظف، وما تحتاج تعدل كودك على أساسها.

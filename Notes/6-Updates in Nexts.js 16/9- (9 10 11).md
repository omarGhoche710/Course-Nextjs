# 9️⃣ Routing / Navigation Internals Overhauled

### شو اللي تغير؟

* **Layout deduplication for prefetch**
  → Next.js صار يقلّل التحميل المكرر للـ layouts أثناء prefetch.
* **Incremental prefetching**
  → prefetch صار تدريجي، يعطي أفضلية لكفاءة نقل البيانات (transfer efficiency).

### شو المعنى العملي؟

* **ما تحتاج تعمل أي تغييرات على الكود.**
* ممكن تشوف **عدد requests أكثر** في DevTools، لكن **البيانات المرسلة أقل**، يعني أسرع تحميل.

> الخلاصة: تحسين داخلي للـ App Router، **شفاف للمطور**.

# 🔟 PPR Model Changed: use cacheComponents

### شو اللي تغير؟

* **experimental_ppr** تم إزالته نهائيًا.
* إذا بدك تستخدم PPR-era behavior (Partial Page Rendering)، لازم تفعل:

```ts
next.config.ts
```

```ts
const nextConfig = {
  cacheComponents: true
};

export default nextConfig;
```

### المعنى العملي

* أي صفحة أو layout عندها **تحديث جزئي** سيعمل الآن باستخدام `cacheComponents`.
* تحسين الأداء وتجربة المستخدم مع تحديثات الصفحة الجزئية.

# 1️⃣1️⃣ Middleware Convention Deprecated → Proxy

### شو اللي تغير؟

* **middleware.ts / middleware.js** تم استبدالها بـ:

```text
proxy.ts / proxy.js
```

* **export** صار:

```ts
export const proxy = ...
```

* بعض config keys القديمة تغيرت لتتوافق مع اسم proxy الجديد.

### Caveat مهم:

* Proxy يستخدم **Node.js runtime فقط**.
* **Edge runtime** لم يعد مدعوم في proxy.

### مثال عملي في مشروعك:

```ts
// app/proxy.ts
import { NextResponse } from "next/server";

export const proxy = (request: Request) => {
  // مثال: إعادة توجيه كل طلبات /api
  if (request.url.includes("/api")) {
    return NextResponse.redirect(new URL("/api-docs", request.url));
  }
  return NextResponse.next();
};
```

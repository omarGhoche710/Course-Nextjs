# 1️⃣ Next/Image Changes (Breaking + Deprecations)

في **Next.js 16** صار عندنا تغييرات كبيرة في `next/image`:

### أهم التغييرات:

1. **Local image query strings**

   * صار لازم تحدد **patterns** للسماح بـ query string على الصور المحلية.
   * مثال: `/next.svg?v=1` يحتاج **allowlist** في `next.config.ts`.

2. **Default minimumCacheTTL**

   * صار 14400s (4 ساعات) بدل 60s.

3. **Default imageSizes**

   * لم يعد يشمل 16، فقط الأحجام الرئيسية.

4. **Default qualities**

   * صار `[75]` بدل القيم القديمة.

5. **Local IP optimization**

   * ممنوع افتراضياً، تحتاج `dangerouslyAllowLocalIP: true` إذا كنت تريد استخدامها.

6. **maximumRedirects**

   * صار 3 بدل أن يكون غير محدود.

7. **next/legacy/image**

   * Deprecated → يجب استخدام `next/image` الحديثة.

8. **images.domains deprecated**

   * استخدم `images.remotePatterns` بدلها.

---

### مثال `next.config.ts` عملي:

```ts id="b7v1sx"
const nextConfig = {
  images: {
    localPatterns: [
      {
        pattern: "**/public/**",
      },
    ],
    remotePatterns: [
      {
        protocol: "https",
        hostname: "example.com",
      },
    ],
    dangerouslyAllowSVG: true,
    dangerouslyAllowLocalIP: false,
    minimumCacheTTL: 14400,
    maximumRedirects: 3,
    imageSizes: [32, 48, 64, 96, 128, 256, 384, 512],
    formats: ["image/avif", "image/webp"],
    quality: [75],
  },
};

export default nextConfig;
```

# 2️⃣ Next dev & build can run concurrently

* **Output directories** منفصلة:

  * Dev → `.next/dev`
  * Build → `.next`
* **Lockfile** يمنع تشغيل نفس الأمر مرتين بنفس المشروع.
* المهم: **لو عندك سكربت داخلي يعتمد على مسار `.next` القديم، حدّثه**.

---

# 3️⃣ Parallel Route Slots now require `default.js/ts(x)`

* أي **parallel slot** (مثلاً `@modal`) صار **لازم يحتوي ملف default.tsx أو default.js**.
* بدون هذا الملف، **البناء يفشل**.

### مثال:

```
app/
  parallel-demo/
    @modal/
      default.tsx  <-- ضروري
```

`default.tsx` ممكن يكون:

```tsx id="xf1f2v"
export default function ModalDefault() {
  return <div>This is the default modal content</div>;
}
```

### 🔹 الخلاصة العملية

| التغيير                   | ماذا تفعل                                                                                |
| ------------------------- | ---------------------------------------------------------------------------------------- |
| next/image                | حدّث `next.config.ts` مع `localPatterns`, `remotePatterns`, `quality`, `minimumCacheTTL` |
| next dev/build concurrent | تحقق من أي سكربت يعتمد على مسار `.next`                                                  |
| Parallel route slots      | تأكد أن كل slot يحتوي `default.tsx` أو `default.js`                                      |

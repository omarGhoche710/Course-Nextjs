## 🟢 شو يعني **cache** أو **caching**؟

**Cache** معناها **تخزين مؤقت**.
يعني بدل ما الكمبيوتر أو السيرفر يحسب الشيء نفسه كل مرة، يخزنه مرة وحدة ويستخدم النسخة المخزنة لاحقًا لتسريع الأداء.

---

### مثال حياتي:

تخيل عندك **صندوق فيه كتب**:

* كل مرة بدك كتاب، بدل ما تروح تشتريه من المكتبة، تروح للصندوق وتاخذه مباشرة.
* هاد الصندوق هو الـ **cache**. يوفر وقت وجهد.

---

### في البرمجة (Next.js):

* **Server Components** صار في Next.js 16 عندهم **cacheComponents: true**
* يعني المكون يخزن النتيجة الأولى في الذاكرة، **ويعيدها بدل ما يعيد الحساب كل مرة**.

مثال:

```tsx
export default function ServerTime() {
  return <div>{new Date().getFullYear()}</div>;
}
```

* قبل cache → كل مرة الصفحة تُعرض → الوقت يُحسب من جديد
* بعد cache → أول مرة تحسب → بعدها يستخدم نفس الوقت المخزن
* **المشكلة**: أي شيء ديناميكي (مثل `new Date()`) رح يعطي خطأ إذا حاولت تستخدمه مباشرة في Server Component بعد التحديث

---

### الحل:

1. أي شيء **ديناميكي** مثل الوقت أو fetch → **Client Component**
2. أي شيء **ثابت** → Server Component مع cache ok

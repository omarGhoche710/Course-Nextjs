# الفكرة العامة للجملة

هو يقول إنهم عندما انتقلوا من استخدام عدة أدوات مع بعض إلى استخدام **Framework واحد** مثل **Next.js** حصلت أشياء مهمة جداً.

---

# قبل (الطريقة القديمة)

كان المشروع مبني باستخدام عدة أدوات:

* React → للـ Frontend
* Express → للـ Backend
* Webpack → لتجميع ملفات المشروع (bundling)

يعني المشروع كان يحتاج **عدة أدوات مختلفة تعمل معاً**.

---

# ماذا حدث بعد استخدام Framework؟

عندما انتقلوا إلى **Next.js**:

### 1️⃣ حذف أكثر من 20,000 سطر كود

يعني الكثير من الكود الذي كانوا يكتبونه بأنفسهم أصبح:

✅ جاهز داخل الـ framework.

---

### 2️⃣ حذف أكثر من 30 مكتبة

كان لديهم:

```
30+ dependencies
```

أي مكتبات إضافية.

لكن بعد استخدام Next.js لم يعودوا بحاجة لها لأن:

**Next.js يوفر هذه الأشياء جاهزة.**

---

### 3️⃣ تحسن سرعة HMR

HMR تعني:

**Hot Module Reloading**

أي عندما تعدل الكود أثناء التطوير، يتم تحديث الصفحة فوراً بدون إعادة تحميل كامل.

السرعة كانت:

```
1.3 ثانية
```

وأصبحت:

```
131ms
```

يعني **أسرع بحوالي 10 مرات** 🚀.

---

# ماذا يعني هذا الكلام؟

الرسالة الأساسية هي:

Framework مثل **Next.js**:

* يقلل الكود
* يقلل المكتبات
* يجعل التطوير أسرع
* يحسن تجربة المطور

---

✅ الخلاصة:

الانتقال من:

```
React + Express + Webpack
```

إلى:

```
Next.js
```

جعل المشروع:

* أبسط
* أسرع
* أسهل في الصيانة

---

💡 معلومة قد تفاجئك:
هناك سبب تقني جعل **Next.js يقلل آلاف الأسطر من الكود** في المشاريع الكبيرة، وهو شيء اسمه **Convention over Configuration** — فكرة قوية جداً في عالم الـ frameworks. إذا أردت أشرحها لك. 🚀

خلّينا نفسّر الفقرة **جملة جملة** حتى تتوضح الفكرة 👇

---

# 1️⃣ لماذا ظهر React ؟

النص يقول:

> React improved the development lifecycle

يعني React جعل **تطوير المواقع أسهل وأسرع**.

وقدّم أشياء مهمّة مثل:

* **Components**
* **Virtual DOM**
* **Client-Server mechanism**

---

# 2️⃣ ماذا يحدث عندما تفتح موقع React؟

عندما تفتح موقع مبني بـ React:

1️⃣ المتصفح يرسل طلب للسيرفر.

```
Browser → Request → Server
```

2️⃣ السيرفر يرسل شيئين رئيسيين:

```
HTML صغير
+
JavaScript كبير (React App)
```

مثال HTML الذي يرسله السيرفر:

```html
<div id="root"></div>
```

هذا **فارغ تقريبًا**.

---

# 3️⃣ ماذا يفعل JavaScript بعد ذلك؟

بعد أن يصل ملف JavaScript:

React يبدأ تشغيل التطبيق داخل المتصفح.

يعني:

```
JavaScript → يبني الصفحة
```

ويسمّى هذا:

**Client Side Rendering**

لأن المتصفح هو الذي يبني الصفحة.

---

# 4️⃣ ما هو الـ Virtual DOM ؟

الـ **DOM** هو شجرة تمثل عناصر الصفحة.

مثال:

```
html
 └ body
    └ h1
```

لكن React لا يغيّر **DOM الحقيقي مباشرة**.

بدل ذلك ينشئ نسخة خفيفة اسمها:

```
Virtual DOM
```

ثم يحدث التالي:

1️⃣ React يغيّر **Virtual DOM**
2️⃣ يقارن التغييرات
3️⃣ يحدّث فقط الأجزاء التي تغيّرت في DOM الحقيقي

هذا يجعل الموقع:

✅ أسرع
✅ أقل تحديثات للصفحة

---

# 5️⃣ مثال بسيط

لو عندك زر يزيد رقم:

قبل الضغط:

```
Count: 0
```

بعد الضغط:

```
Count: 1
```

React لا يعيد بناء الصفحة كلها.

هو يغيّر فقط:

```
0 → 1
```

---

# 6️⃣ خلاصة الفكرة

في مواقع React:

السيرفر يرسل:

```
HTML بسيط
+
JavaScript
```

ثم:

```
JavaScript (React)
↓
يبني الصفحة في المتصفح
↓
يستعمل Virtual DOM للتحديث السريع
```

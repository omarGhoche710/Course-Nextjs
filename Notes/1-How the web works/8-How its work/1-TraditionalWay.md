أكيد. خلّيني اشرحلك الفكرة **ببساطة خطوة خطوة** 👇

النص عم يحكي عن **كيف كان الموقع يشتغل تقليديًا (Traditional Websites)** بين **المتصفح (Browser)** و **السيرفر (Server)**.

---

## 1️⃣ المستخدم يطلب صفحة

لما أنت تفتح موقع مثلًا:

```
www.example.com
```

المتصفح تبعك يبعث **request (طلب)** للسيرفر.

---

## 2️⃣ السيرفر يرد بثلاث ملفات

السيرفر يرسل عادةً:

* **HTML** → هيكل الصفحة
* **CSS** → شكل وتصميم الصفحة
* **JavaScript** → التفاعل والوظائف

مثال:

```
index.html
style.css
script.js
```

---

## 3️⃣ المتصفح يقرأ HTML أولاً

المتصفح يبدأ بقراءة **HTML** لأنه يحتوي على **هيكل الصفحة**.

مثال:

```html
<h1>Hello</h1>
<button>Click</button>
```

يعرف أن هناك:

* عنوان
* زر

---

## 4️⃣ يطبق CSS

بعدها المتصفح يطبّق **CSS** ليعطي الشكل.

مثال:

```css
h1 {
  color: red;
}
```

فيصبح العنوان **أحمر**.

---

## 5️⃣ يشغّل JavaScript

إذا يوجد **JavaScript** المتصفح يشغّله ليضيف التفاعل.

مثال:

```javascript
button.addEventListener("click", () => {
  alert("Hello");
});
```

النتيجة:

عندما تضغط الزر يظهر **alert**.

---

## 6️⃣ ماذا يحدث عند تغيير الصفحة؟

إذا ضغطت رابط مثل:

```
/about
```

المتصفح يرسل **طلب جديد للسيرفر**.

السيرفر يرسل مرة أخرى:

```
about.html
about.css
about.js
```

ويعيد المتصفح تحميل الصفحة بالكامل.

---

## 7️⃣ ملخص الفكرة

الطريقة التقليدية تعمل هكذا:

```
User -> Request -> Server
Server -> HTML + CSS + JS -> Browser
Browser -> Render Page
```

وعند الانتقال لصفحة جديدة:

```
Request جديد
تحميل جديد كامل للصفحة
```

---

💡 لهذا السبب ظهرت تقنيات مثل **React** و **Next.js** لتقليل إعادة تحميل الصفحة وجعل المواقع أسرع.

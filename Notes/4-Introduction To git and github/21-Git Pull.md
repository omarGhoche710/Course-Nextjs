---

# 🧠 شو يعني `git pull`؟

`git pull` = **تجيب آخر التحديثات من GitHub أو أي remote repository** وتدمجها مع الفرع الحالي عندك محليًا.

بمعنى آخر:

```text
GitHub (remote) → Local (لابتوبك)
```

---

# 🎯 كيف بيشتغل؟

`git pull` = عمليتين في واحد:

1. **fetch** → بيجيب كل التغييرات من الريموت (لكن ما يدمجها)
2. **merge** → بدمج التغييرات مع الفرع الحالي عندك

---

# 🔹 مثال عملي

انت على branch `main` عندك 5 commits

وحدا من التيم عمل 2 commits على GitHub

```bash
git pull
```

بعدها:

* commits الجديدة (2 commits) من GitHub بتنضاف عندك
* صار عندك كل history كامل (5 + 2) ✅

---

# ⚠️ ملاحظات مهمة

1. **لازم تكون branch الحالي نفس اسم الريموت**

   * مثلا `main` local ↔ `origin/main`

2. إذا صار conflict بين شغلك وشغل التيم:

   * Git رح يطلب منك تحل المشكلة قبل ما تخلص pull

3. `git pull` بيشتغل **بس بعد ما يكون في ربط مع origin**

   * يعني لازم تكون عملت push أو `git remote add origin ...` قبل

---

# 🧠 الفكرة

`git pull` **دايمًا يعمل pull من الفرع المقابل على الريموت** يلي انت واقف عليه محليًا.

---

## مثال:

1️⃣ انت واقف على:

```bash
* main
```

* التيم عدّل على `main` على GitHub
* تعمل:

```bash
git pull
```

✔ رح يجيب آخر التحديثات من `origin/main` ويدمجها عندك

---

2️⃣ انت واقف على:

```bash
* feature-login
```

* التيم عدّل على `feature-login` على GitHub
* تعمل:

```bash
git pull
```

✔ رح يجيب آخر التحديثات من `origin/feature-login` ويدمجها عندك

---

# 💡 جملة تحفظها

> **Git دايمًا يعرف انت على أي branch محلي، وبيجيب التحديثات من نفس الفرع على GitHub** ✅

---

بالضبط صح 💯🔥

`git pull` **ما يمسح commits عندك**، بس بيجيب **الجديد فقط من الريموت** وبيدمجهم مع شغلك المحلي.

---

# 🧠 مثال بصري

## قبل:

```text
Local main:
A --- B --- C

Remote main (GitHub):
A --- B --- C --- D --- E
```

---

## بعد `git pull`:

```text
Local main:
A --- B --- C --- D --- E
```

* commits **D و E** جديدة من الريموت
* commits عندك (A, B, C) **بتضل موجودة** ✅

---

# ⚠️ ملاحظات مهمة

1. لو عندك تغييرات محلية **ما عملت commit لها** قبل pull:

   * Git رح يرفض أو يطلب منك stash أو commit قبل ما يدمج

2. إذا صار conflict بين commits عندك والـ commits الجديدة:

   * Git رح يطلب منك تحل التعارض قبل إكمال pull

---

# 💡 خلاصة

> **`git pull` = يجيب كل التغييرات الجديدة من الريموت بدون حذف شغلك المحلي**

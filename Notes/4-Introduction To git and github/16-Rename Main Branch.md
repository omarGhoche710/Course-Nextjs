تمام 👌 هيدا أمر مهم وكتير بينستخدم 🔥

# 🧠 شو يعني:

```bash
git branch -M main
```

👉 يعني:
**غيّر اسم الـ branch الحالي إلى `main`**

---

# 🔍 نفكّكها

## 🟢 `git branch`

👉 أمر للتعامل مع الـ branches

---

## 🟡 `-M`

👉 اختصار لـ:
**Move / Rename (إعادة تسمية)**

⚠️ و capital `-M` يعني:
👉 **حتى لو في branch بنفس الاسم، غصّبًا بدو يستبدلو**

---

## 🔵 `main`

👉 الاسم الجديد للـ branch

---

# 🎯 مثال عملي

انت عندك:

```id="wq6c9l"
master
```

بدك تغيّر الاسم لـ `main`:

```bash id="x6y4yl"
git branch -M main
```

👉 بصير:

```id="g9k3v9"
main
```

---

# 🤔 ليش منعمل هيك؟

قديماً Git كان يستخدم:
👉 `master`

هلق صار standard:
👉 `main`

فكتير مشاريع بتعمل rename

---

# 🧠 متى تستعملها؟

* بعد `git init`
* أو قبل ما تربط مع GitHub
* أو لما بدك توحّد أسماء الفروع

---

# ⚠️ الفرق بين -m و -M

| الأمر | شو بيعمل                   |
| ----- | -------------------------- |
| `-m`  | rename عادي                |
| `-M`  | rename غصب حتى لو في تعارض |

---

# 🧠 الخلاصة

👉 `git branch -M main`
= **غيّر اسم الفرع الحالي لـ main حتى لو في مشاكل**

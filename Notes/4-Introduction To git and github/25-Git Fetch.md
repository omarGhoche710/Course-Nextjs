تمام 🔥 هيدا من أهم المفاهيم يلي بيفرّق بين شخص مبتدئ ومحترف 👌

# 🧠 شو يعني `git fetch`؟

👉 **`git fetch` = يجيب آخر التحديثات من GitHub بدون ما يدمجها مع شغلك**

---

# 🎯 الفرق الأساسي

```text
fetch = يجيب بس
pull  = يجيب + يدمج
```

---

# 🧪 مثال

## قبل:

```text
Local main:
A --- B --- C

Remote main:
A --- B --- C --- D --- E
```

---

## إذا عملت:

```bash
git fetch
```

👉 شو بصير؟

* بيجيب D و E
* بس ما بيضيفهم لملفاتك

```text
Local main:
A --- B --- C

origin/main:
A --- B --- C --- D --- E
```

---

# 🤯 الفكرة المهمة

👉 بعد `fetch`:

* فيك تشوف التغييرات
* تقرر إذا بدك تدمج أو لا

---

# 🔥 كيف تستفيد منه؟

## 1️⃣ تشوف الفرق:

```bash
git log main..origin/main
```

👉 بيعطيك commits الجديدة

---

## 2️⃣ إذا عجبك الوضع:

```bash
git merge origin/main
```

---

# ⚠️ ليش نستخدمه بدل pull؟

لأنو:

👉 `pull` بيدمج دغري
👉 `fetch` بيعطيك تحكّم أكتر 😏

---

# 🎮 مثال بالحياة

👤 رفيقك عدّل كود
بدك تشوف شو عمل بدون ما تخرب شغلك:

```bash
git fetch
```

👀 بتراجع
بعدين بتقرر تدمج أو لا

---

# 💡 جملة تحفظها

> **`fetch` = تحديث معلوماتك عن الريموت بدون لمس شغلك**

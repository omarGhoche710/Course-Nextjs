تمام 🔥 هيدي أهم نقطة بالتيم — ركّز معي شوي 👇

---

# 🧠 شو يعني Pull Request (PR)؟

👉 **Pull Request** = طلب دمج شغلك مع branch تاني (غالبًا `main`)

يعني:

> “يا جماعة، خلّصت شغلي، فيكن تراجعوه وتدمجوه؟”

---

# 🎯 مثال بسيط

1️⃣ انت اشتغلت على branch:

```bash
feature-login
```

2️⃣ خلصت ورفعت:

```bash
git push
```

3️⃣ بتروح على GitHub
وبتعمل **Pull Request**

---

# 🧠 شو بصير داخل PR؟

لما تعمل PR:

👀 الفريق بيشوف:

* شو الكود يلي كتبته
* شو التعديلات
* الفرق بينه وبين `main`

---

## ✍️ بعدين:

* حدا من التيم يعمل **review**
* يعطيك ملاحظات
* يمكن يقولك:

  * عدّل هون
  * في bug هون

---

## ✅ إذا كل شي تمام:

👉 بينعمل:

```text
Merge
```

يعني:

```text
feature-login → main
```

---

# 🎮 تخيّلها هيك

```text
main (مستقر)
   \
    feature-login (شغلك)
```

PR = جسر بيناتهم 🌉

---

# 🤯 ليش PR مهم؟

❌ بدون PR:

* أي حدا ممكن يكسر المشروع 😅

✅ مع PR:

* في مراجعة
* في تنظيم
* في أمان
* في تتبع لكل تغيير

---

# 💡 جملة تحفظها

> **Pull Request = طلب دمج + مراجعة قبل الدمج**

---

# 🧠 جواب سؤالك قبل شوي 👀

👉 إذا انت وزميلك على نفس branch:

❌ غالبًا ما في PR
(وهيدا غلط بالتيمات الكبيرة)

✅ الأفضل:

* كل واحد branch
* وبعدين PR

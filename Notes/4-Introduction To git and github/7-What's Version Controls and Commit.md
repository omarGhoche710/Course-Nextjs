تمام 🔥 هيدا أهم أساس بـ Git، إذا فهمتهن بتصير الأمور كلها سهلة 👇

---

## 🟢 أول شي: شو هو Version Control؟

**Version Control** يعني:

👉 نظام بيخلّيك **تتابع كل التعديلات** يلي بتعملها على الكود

---

### 💡 الفكرة ببساطة:

تخيّل عم تشتغل على مشروع:

* Version 1 → أول نسخة
* Version 2 → عدّلت شي
* Version 3 → ضفت feature

👉 كل نسخة محفوظة، وبتقدر ترجع لأي وحدة

---

### 🧠 ليش مهم؟

* إذا خرب الكود → بترجع لنسخة قديمة
* بتعرف مين عدّل شو (بالتيم)
* ما بتضيع شغلك

---

### 📦 مين بيدير هالشي؟

👉 **Git**

---

## 🟣 تاني شي: شو هو Commit؟

**Commit** يعني:

👉 **حفظ نسخة من المشروع بحالة معيّنة**

---

### 💡 مثال:

أنت اشتغلت وخلصت navbar

بتعمل:

```bash
git commit -m "added navbar"
```

👉 هيك قلت لـ Git:

> “احفظلي هالحالة من المشروع”

---

### 🧠 فكّر فيها هيك:

* commit = صورة (snapshot 📸) للمشروع
* كل commit = مرحلة من شغلك

---

## 🔁 العلاقة بينهم

* Version Control = النظام الكامل
* Commit = عملية الحفظ داخل هالنظام

---

## 🎬 مثال من الحياة

تخيّل:

* عم تكتب مشروع Word

بدل:

* file_final.doc
* file_final_v2.doc
* file_final_last.doc 😅

👉 Git بيعمل:

* commits مرتّبين
* وكل نسخة محفوظة بدون لخبطة

---

## 🔥 الخلاصة

### Version Control:

👉 نظام لتتبع التعديلات

### Commit:

👉 حفظ نسخة من المشروع

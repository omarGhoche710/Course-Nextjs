تمام 🔥 خلّيني أعطيك **سيناريو حقيقي** تمشي عليه خطوة خطوة لتفهم الفكرة صح:

---

## 🎬 السيناريو: أنت عم تعمل مشروع موقع

تخيّل إنك عم تعمل موقع اسمه:
👉 `my-website`

---

## 🟢 المرحلة 1: تشتغل لحالك

بتبلّش تكتب كود على جهازك:

* index.html
* style.css

هلق بتعمل:

👉 تستخدم **Git**

* `git init` → عملت repository
* `git add .`
* `git commit -m "first version"`

💡 هلق صار عندك نسخة محفوظة من المشروع

---

## 🟢 المرحلة 2: بدك تحفظه أونلاين

بتروح على **GitHub**
وتعمل Repository جديد

بعدين:

* `git remote add origin ...`
* `git push`

💡 هيك صار مشروعك موجود أونلاين

---

## 🟢 المرحلة 3: كملت شغل

بتضيف feature جديدة (مثلاً navbar)

بتعمل:

* `git add .`
* `git commit -m "added navbar"`
* `git push`

💡 صار عندك history:

* نسخة 1
* نسخة 2

---

## 🟢 المرحلة 4: خرب الكود 😅

عملت تعديل وغلطت

👉 بدل ما تتوتر:

* بترجع لنسخة قديمة (commit)

💡 هون بتفهم قوة Git 🔥

---

## 🟢 المرحلة 5: صار عندك رفيق يشتغل معك

رفيقك بيعمل:

* clone للمشروع من GitHub

هو بيشتغل على feature لحاله
وأنت على feature تانية

---

## 🟢 المرحلة 6: الدمج (أهم شي)

رفيقك بيعمل:
👉 Pull Request

أنت:

* بتراجع الكود
* إذا تمام → merge

💡 هيك بتشتغلوا بدون ما تخربوا على بعض

---

## 🧠 الخلاصة من السيناريو

* Repository = المشروع
* Git = بيدير التعديلات
* GitHub = بيخلي المشروع أونلاين
* Commit = حفظ نسخة
* Push = رفع
* Pull = تنزيل
* Branch = تجربة شي جديد
* Pull Request = طلب دمج

# 🧠 أول شي: شو هي staging area؟

الـ **staging area** هي:
👉 مكان مؤقّت بتحط فيه التعديلات قبل ما تعمل commit

يعني:

* Git ما بياخد كل شي دغري
* انت بتقرّر شو بدك تحفظ

---

# 🎮 تخيّلها هيك (كتير مهم)

عندك 3 مراحل:

## 1️⃣ Working Directory

👉 هون عم تشتغل (تعدّل ملفاتك)

## 2️⃣ Staging Area

👉 هون بتحط الأشياء اللي **بدك تحفظها**

## 3️⃣ Commit

👉 هون بتنحفظ رسميًا 📸

---

# 🔁 الفلو الكامل

## ✏️ 1. عدّلت ملفات

```bash
git status
```

➡️ modified

---

## ➕ 2. اخترت شو بدك تحفظ

```bash
git add navbar.js
```

👉 هون دخل على staging area

---

## 💾 3. حفظت

```bash
git commit -m "fix navbar"
```

👉 صار snapshot

---

# 🤯 أهم فكرة

👉 **ليش في add قبل commit؟**

لأنو Git بيعطيك تحكّم 💪

---

## 🎯 مثال قوي

انت عدّلت 3 ملفات:

* navbar.js ✅
* footer.js ❌ (فيه غلط)
* style.css 🤷‍♂️

بدك تحفظ بس navbar:

```bash
git add navbar.js
git commit -m "fix navbar"
```

👉 هيك:

* حفظت الصح ✅
* تركت الغلط ❌

---

## ❌ لو ما في staging؟

كان:
👉 يا بتاخد كل شي
👉 يا ولا شي

وهيدا سيّء كتير 😅

---

# 🧠 خلاصة ذهبية

| المرحلة | شو بتعمل |
| ------- | -------- |
| Working | عم تشتغل |
| Staging | عم تختار |
| Commit  | عم تحفظ  |

---

# 💡 جملة تحفظها:

👉 **git add = أنا قررت شو بدي**
👉 **git commit = أنا حفظت القرار**

---

إذا بدك نعمل تمرين عملي 🔥
بعطيك سيناريو (فيه 3 ملفات) وانت بتقرر شو تعمل step by step 😏

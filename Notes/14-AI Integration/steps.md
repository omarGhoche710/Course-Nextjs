- UseChat:
  ```js 
  const { messages, sendMessage, status } = useChat({
    transport: new DefaultChatTransport({
      api: 'api/chat',
    }),
  });
  ```
      هذه الشيفرة (Code) مكتوبة بلغة **JavaScript** وتحديداً باستخدام مكتبة **React**، وهي تهدف إلى إعداد نظام محادثة (Chat) في تطبيقك.

    إليك شرح ما تفعله هذه الأسطر باللغة العربية الفصحى:

    ### الشرح العام
    هذا الكود يقوم باستدعاء "خطاف مخصص" (Custom Hook) يُسمى `useChat`. وظيفة هذا الخطاف هو تولي كل العمليات المعقدة المتعلقة بالمحادثة بدلاً منك، مثل إرسال الرسائل، استقبالها، ومعرفة حالة الاتصال.

    ---

    ### تفصيل المكونات:

    **1. المخرجات (`{ messages, sendMessage, status }`):**
    هذه هي الأدوات التي يمنحك إياها المحرك لكي تستخدمها في واجهة المستخدم:
    * **`messages`**: هي مصفوفة (Array) تحتوي على جميع الرسائل التي تمت كتابتها (سواء من المستخدم أو الطرف الآخر) لعرضها على الشاشة.
    * **`sendMessage`**: هي دالة (Function) تستخدمها عندما يضغط المستخدم على زر "إرسال" لبعث نص جديد.
    * **`status`**: متغير يخبرك بحالة المحادثة الحالية (مثلاً: "يتم الإرسال"، "متصل"، "خطأ").

    **2. الإعدادات (`transport`):**
    هنا أنت تحدد للمحرك **كيف** و **أين** سيتواصل مع الخادم (Server):
    * **`DefaultChatTransport`**: هو القالب المسؤول عن عملية نقل البيانات.
    * **`api: 'api/chat'`**: هو العنوان (URL) أو الرابط الذي سيتم إرسال الرسائل إليه في الخلفية (Backend) لكي تتم معالجتها.

    ---

    ### الخلاصة:
    ببساطة، أنت تقول للتطبيق: *"يا React، أريد استخدام نظام محادثة يتصل بهذا الرابط (`api/chat`)؛ أعطني قائمة بالرسائل، وطريقة لإرسال رسائل جديدة، وأخبرني إذا كان هناك عطل في الاتصال."*

- Submit Function and use SendMessage:
  ```jsx
    const submit  = (text: string) => {
    if (!text.trim()) return;
    sendMessage({ parts: [{ type: 'text', text }] });
    setInput("");
  }
  ```
  **وظيفة دالة الـ `submit` في نظام المحادثة:**

  1.  **التحقق (Validation):** تستخدم `.trim()` للتأكد من أن النص ليس فارغاً أو مجرد مسافات؛ إذا كان فارغاً، تتوقف الدالة فوراً.
  2.  **الإرسال (Sending):** تستدعي `sendMessage` وتُرسل النص داخل مصفوفة `parts` (هذا التنسيق يسمح بإضافة صور أو ملفات مستقبلاً).
  3.  **التنظيف (Reset):** تقوم بتفريغ حقل الإدخال (`setInput("")`) بعد الإرسال مباشرة لتهيئته للرسالة التالية.

  **باختصار:** هي المسؤولة عن "فلترة" النص، إرساله، ثم تنظيف الشاشة.

* get message text function:
   ```tsx
    const getMessageText = (parts: { type: string, text?: string }[]) => {
      const textPart = parts.find(part => part.type === "text");

      if (textPart && textPart.text) {  
        return textPart.text;
      }

      return parts.length > 0 ? "[محتوى غير نصي]" : "";
    }
   ```
   إليك ملاحظة ملخصة (Note) تشرح هذا الكود والسبب التقني وراء استخدام `?` في التعريف:

    ### ملخص كود `getMessageText`
    هذه الدالة تعمل كـ **"مستخرج آمن"** للنص من الرسالة. هي تبحث داخل مصفوفة الأجزاء (`parts`) عن أول جزء من نوع `"text"`. إذا وجدته وكان يحتوي على نص فعلي، تقوم بإرجاعه؛ وإذا كانت الرسالة تحتوي على أشياء أخرى (مثل صور أو أدوات) بدون نص، تُعلمك بوجود "[محتوى غير نصي]".

    ---

    ### لماذا وضعنا `text?: string`؟ (علامة الاستفهام)

    في TypeScript، علامة الاستفهام تعني أن هذه الخاصية **اختيارية (Optional)**. وهذا ضروري جداً هنا للأسباب التالية:

    1.  **اختلاف أنواع الأجزاء:** مصفوفة الـ `parts` تحتوي على أنواع مختلفة. جزء النص لديه خاصية `text` فعلياً، لكن "جزء الصورة" أو "جزء الأداة" لا يحتوي على مفتاح `text` بل يحتوي على مفاتيح أخرى (مثل `url` أو `toolName`).
    2.  **تجنب الأخطاء:** لو لم نضع `?` لظن TypeScript أن **كل** جزء في المصفوفة (حتى الصور) "يجب" أن يحتوي على نص، وهذا غير صحيح وسيؤدي لخطأ برمي.
    3.  **التعامل مع الـ `undefined`:** بوضع `?` نحن نخبر الكود: "ابحث عن `text`؛ إذا وجدته كان بها، وإذا لم تجده (كان `undefined`) فلا مشكلة، تعامل مع الأمر بهدوء".



    ### شرح الـ `if (textPart && textPart.text)`
    * **`textPart`**: تتأكد أولاً أن عملية البحث (`find`) نجحت ولم تكن النتيجة "لا شيء".
    * **`textPart.text`**: تتأكد ثانياً أن النص الموجود داخل هذا الجزء ليس فارغاً أو `undefined`.

    **باختصار:** علامة الاستفهام هي "صمام أمان" يخبر TypeScript أن محتوى الرسالة متنوع وليس دائماً عبارة عن نص فقط.

* api/chat:

إليك شرح مفصل لهذا الكود، والذي يعمل كـ **"جسر"** بين المتصفح وبين عقل الذكاء الاصطناعي (Claude). وظيفته الأساسية هي استقبال سؤالك، إرساله للموديل، وإرجاع الإجابة "مقطرة" (Streaming) لكي تظهر فوراً.


### 1. الاستيراد (The Imports)
* **`anthropic`**: هو "الموزع" الذي يسمح للكود بالاتصال بشركة Anthropic.
* **`streamText`**: هذه هي الأداة السحرية التي تجعل النص يظهر كلمة بكلمة (مثل ChatGPT) بدلاً من انتظار الإجابة كاملة.
* **`convertToModelMessages`**: وظيفتها "ترجمة" شكل الرسائل من التنسيق الذي يفهمه المتصفح (UI) إلى التنسيق التقني الذي يفهمه Claude.

---

### 2. الإعدادات (Configuration)
```typescript
export const maxDuration = 30;
```
هذا السطر يخبر السيرفر (مثل Vercel): "مسموح لك أن تستغرق 30 ثانية كحد أقصى لتوليد الإجابة قبل أن تعتبر الطلب قد فشل". هذا ضروري لأن الـ AI قد يستغرق وقتاً في التفكير.

---

### 3. الدالة الأساسية (`POST` Function)
هذه الدالة تعمل عندما يرسل المتصفح طلباً (Request) إلى هذا المسار:

1.  **استلام البيانات:**
    ```typescript
    const { messages } = await request.json();
    ```
    هنا نقوم بفتح "الطرد" القادم من المتصفح لاستخراج مصفوفة الرسائل (`messages`) التي تحتوي على كل ما دار في المحادثة حتى الآن.

2.  **استدعاء الموديل (`streamText`):**
    هنا نحدد 3 أشياء أساسية للموديل:
    * **`model`**: حددنا الشخصية التي ستجيب، وهي هنا `claude-haiku-4-5-20251001`.
    * **`system`**: هنا نضع "قواعد اللعبة". أخبرناه أنه مساعد للمبرمجين المبتدئين، وهذا سيجعل إجاباته مبسطة وواضحة.
    * **`messages`**: أرسلنا له تاريخ المحادثة بالكامل لكي يعرف عما نتحدث.



---

### 4. إرجاع النتيجة (The Response)
```typescript
return result.toUIMessageStreamResponse();
```
هذا السطر هو الذي "يفتح الحنفية". هو لا يرسل نصاً عادياً، بل يرسل **Stream** (تدفق بيانات). المتصفح يستقبل هذا التدفق ويقوم بتحديث الشاشة أمامك فور وصول كل حرف جديد.

---

### ملخص العملية (Note)
تخيل أن هذا الكود هو **"مترجم فوري"**:
1.  يسمع ما تقوله (`request.json`).
2.  يصيغه بشكل رسمي ويوضح للمتحدث (Claude) طبيعة عمله (`system`).
3.  ينقل الكلام لـ Claude.
4.  يأخذ الإجابة من Claude ويعيد قولها لك حرفاً بحرف دون تأخير (`toUIMessageStreamResponse`).

**لماذا هذا الكود احترافي؟**
* لأنه يستخدم **Streaming**، مما يحسن تجربة المستخدم.
* لأنه يفصل بين "واجهة المستخدم" وبين "مزود الذكاء الاصطناعي"، مما يسهل تغيير الموديل مستقبلاً.


# 📌 Next.js Rendering & Caching (Production Mode)

بشكل عام، Next.js بيحاول يكون "ذكي" ويقدم أسرع أداء ممكن عن طريق التمييز بين نوعين من الصفحات:

### 1. الـ Static Rendering (الوضع الافتراضي)
* **شو بصير؟** Next.js بيعمل "رندرة" (Render) للصفحة **مرة واحدة بس** وقت الـ `build` (لما تعمل `npm run build`).
* **الـ Cache:** البيانات بتتحول لملفات HTML و JSON ثابتة، وبتتخزن (Cached) على السيرفر أو الـ CDN.
* **النتيجة:** لما المستخدم يطلب الصفحة، بتوصله فوراً بدون ما السيرفر يرجع يعيد تشغيل الكود.
* **متى بستخدمه؟** للصفحات اللي بياناتها ما بتتغير كتير (مثل: About Us، Blog Post، Documentation).



### 2. الـ Dynamic Rendering
* **شو بصير؟** Next.js بيعمل رندرة للصفحة **عند كل طلب** (Request Time).
* **الـ Cache:** بشكل افتراضي، **ما بصير في Cache** للبيانات الديناميكية لأنها بتتغير حسب المستخدم أو الوقت.
* **المحفزات (Triggers):** أي صفحة بتستخدم هول الأشياء بتتحول تلقائياً لـ Dynamic:
    1. **Dynamic Functions:** مثل `cookies()` أو `headers()`.
    2. **Search Params:** لما الصفحة تعتمد على الـ URL Query (مثل `?id=123`).
    3. **No-Store Fetch:** إذا كتبت بيدك `{ cache: 'no-store' }`.

---

### 💡 كيف بتعرف الفرق بالـ Terminal؟
لما تعمل `npm run build` بالـ VS Code، انتبه للرموز اللي بتطلع بآخر الـ Build:

* **`○` (Circle):** تعني **Static** (يعني Next.js عمل لها Cache ومارح تتغير).
* **`ƒ` (Function):** تعني **Dynamic** (يعني السيرفر رح يشغل الكود مع كل زيارة).

---

### 🚀 قاعدة ذهبية للمبرمج:
* **Static** = سرعة خارقة + SEO ممتاز + توفير بموارد السيرفر.
* **Dynamic** = بيانات حية + مخصصة لكل مستخدم + استهلاك أكتر للسيرفر.

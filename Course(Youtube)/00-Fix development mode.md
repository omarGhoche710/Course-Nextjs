### 💡 خطوات الـ Production Build الناجح (Next.js)

**1. تنظيف الـ Cache (إذا في تعليق أو أخطاء غريبة):**
بما إنك بتستخدم PowerShell، استعمل هيدا الأمر لتمسح الفولدر القديم وتضمن إنك عم تبني من الصفر:
```powershell
Remove-Item -Recurse -Force .next
```

**2. بناء المشروع (Build):**
هيدي الخطوة بتحول الكود لملفات سريعة وجاهزة للتشغيل:
```bash
npm run build
```
* **تأكد:** لازم تخلص العملية ويعطيك جدول الـ Routes (الدوائر ○ والـ ƒ). إذا طلع Error هون، ما فيك تعمل Start.

**3. تشغيل السيرفر (Start):**
بعد نجاح الـ Build، شغل المشروع بوضع الإنتاج:
```bash
npm run start
```

---

### ⚠️ حلول سريعة للمشاكل المتكررة:

* **مشكلة الـ Port 3000 (العنوان مستخدم):**
    إذا قلك `EADDRINUSE` جرب تشغله على Port تاني فوراً:
    ```bash
    npx next start -p 3005
    ```

* **تخطي أخطاء الـ TypeScript و ESLint:**
    إذا كنت مستعجل وبدك تعمل Build رغم وجود أخطاء بسيطة بالـ Types، تأكد إن ملف `next.config.ts` فيه هول الأسطر:
    ```javascript
    typescript: { ignoreBuildErrors: true },
    eslint: { ignoreDuringBuilds: true }
    ```

* **الفرق بين الأوامر:**
    * `npm run dev`: للشغل والتعديل (بطيء بالمتصفح بس سريع بالتحديث).
    * `npm run build`: لتجهيز النسخة النهائية (مرة واحدة بعد ما تخلص كود).
    * `npm run start`: لتشغيل النسخة اللي جهزتها بالـ Build (سريع جداً).

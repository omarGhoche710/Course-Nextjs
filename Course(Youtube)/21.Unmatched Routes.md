# Unmatched Routes

Navigation from the UI
When navigating through the UI (like clicking links), Next.js keeps showing
whatever was in the unmatched slots before
Page reload

Next.js looks for a `default.tsx' file in each unmatched slot

This file is critical as it serves as a fallback to render content when the framework
cannot retrieve a slot's active state from the current URL

- Example: 
  - we have archived folder inside @notifications => @notifications/archived
  - we have link inside the slot @notifications wich navigate to complex-dashboard/archived
  and we have link inside the folder archived wich navigate to complex-dashboard
  - when we navigate to each page, no error occured...
  - when we reload the page => page not found... that's because the other slots
  become unmatched routes in the url("complex-dashboard/archived").

- Solution:
  - we use in each slot a predefine file called [default.js].x`
